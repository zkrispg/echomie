# EchoMie 微信小程序上线需求清单

> 用途：梳理小程序正式发布前需完成的合规、技术、运营与验收项。执行人可在「状态」栏填写：未开始 / 进行中 / 已完成 / 不适用。

---

## 一、资质与合规（必做）

| 序号 | 需求项 | 说明 | 责任人 | 状态 |
|------|--------|------|--------|------|
| C-01 | 主体资质 | 小程序主体与阿里云账号、域名持有者建议一致或关系清晰，便于备案与审核 | | |
| C-02 | ICP 备案 | 国内服务器提供 Web/API 服务需完成 **ICP 备案**；备案主体与小程序主体一致或符合微信要求 | | |
| C-03 | 域名 | 已注册并完成 **实名认证** 的域名（如 `.top` 等，以微信后台是否可添加为准） | | |
| C-04 | HTTPS | 后端 API 必须使用 **HTTPS**，证书有效（建议 Let's Encrypt / 云厂商证书） | | |
| C-05 | 隐私政策 | 提供可公网访问的 **《隐私政策》** 链接（小程序后台「用户隐私保护指引」配置） | | |
| C-06 | 用户协议（建议） | 提供 **《用户服务协议》** 链接，涉及账号、内容、责任条款 | | |
| C-07 | 个人信息收集说明 | 按微信规范声明收集字段（如 openid、昵称、头像、上传图片等）及用途 | | |

## 二、微信侧配置（必做）

| 序号 | 需求项 | 说明 | 责任人 | 状态 |
|------|--------|------|--------|------|
| W-01 | 小程序 AppID | 已在微信公众平台注册小程序，获取 AppID | | |
| W-02 | 服务器域名 | 在「开发 → 开发管理 → 服务器域名」配置 **request 合法域名**（仅 HTTPS，需备案域名） | | |
| W-03 | uploadFile / downloadFile | 若使用上传、下载文件接口，分别配置 **uploadFile、downloadFile** 合法域名（可与 API 同域） | | |
| W-04 | 业务域名（按需） | WebView 内嵌 H5 时需配置 **业务域名** 并完成校验文件 | | |
| W-05 | AppSecret 安全 | AppSecret **仅放服务端**，勿写入小程序前端代码或 Git 仓库 | | |
| W-06 | 类目与资质 | 选择小程序 **服务类目**，若涉及特殊行业上传对应资质 | | |
| W-07 | 用户隐私保护 | 在后台完成 **用户隐私保护指引** 及接口声明（录音、相册等） | | |

## 六、验收标准（上线前勾选）

- [ ] 使用 **正式域名 + HTTPS**，在真机（非仅开发者工具）完成登录与一次完整上传→查看卡片流程  
- [ ] 微信后台 **服务器域名** 已配置且保存成功，无红色报错  
- [ ] **隐私政策** URL 可公网打开，内容与小程序实际收集信息一致  
- [ ] 备案号展示（若类目或当地要求展示 ICP 备案号，在关于页或 foot 展示）  
- [ ] 体验成员签字确认 **验收通过**  

---

## 七、风险与依赖说明

1. **未备案域名**：国内 ECS 上 API 可能被拦截，小程序 **无法** 将未备案域名加入 request 合法域名。  
2. **仅 IP 访问**：小程序 request **不支持** 裸 IP 作为合法域名，必须域名 + HTTPS。  
3. **EchoMie 与网页版**：后端一套、小程序与 Web 可并行；小程序额外依赖 **微信登录** 与 **合法域名**。
